# Case Study Library

This is the internal case study repository. Each completed engagement should produce a case study using the template in the toolkit repo (`templates/case-study-template.md`).

---

## Purpose

Case studies serve three functions:

1. **Portfolio evidence** — prove the methodology works to future clients
2. **Methodology refinement** — document what worked, what didn't, and what to adjust
3. **Grant applications** — provide concrete evidence for funder proposals

---

## Case Study Template

Each case study should cover:

### Context

- Client/organization description (anonymize if no permission)
- What triggered the engagement (pain point, event, referral source)
- Engagement tier (Founder Story / Program Engagement / Organizational Embedding)
- Timeline and scope

### The Work

- What phases were activated (Container → Excavation → Performance → Translation)
- Key facilitation moments — what shifted
- Methodology adaptations made for this specific context
- Camera-shipping model used? If yes, what was the participant experience?

### Bilingual Dashboard Outcomes

**Royal Metrics:**

- Quantitative outcomes (completion, satisfaction, utilization, etc.)
- Comparison to targets set during Engagement Design (Stage 4)

**Nomadic Indicators:**

- Reciprocity shifts observed
- Narrative coherence changes
- Somatic safety signals
- Surprise emergence — what nobody planned for

### Synthesis

- What do both registers tell us when read together?
- Where did they converge? Diverge?
- What would you do differently next time?

### Quotes (with permission)

- 1–2 direct quotes from the client that capture the experience
- Attribution format: role/context, not name (unless explicitly approved)

## Case Study Workflow

1. **During engagement:** Take notes throughout, especially during facilitation sessions
2. **At delivery:** Ask for portfolio permission and quote approval
3. **Within 2 weeks of offboarding:** Write the case study using the template
4. **Store:** Obsidian vault (`Practice/Rubinstein Productions/Case Studies/`)
5. **Publish:** Adapt for website, LinkedIn, and grant applications as appropriate
