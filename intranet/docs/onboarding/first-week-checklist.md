# First Week Checklist

Whether you're a new collaborator or Isaac revisiting the system after time away, this is the orientation sequence. Work through it in order.

---

## Day 1: Understand the Practice

- [ ] Read [Mission & Positioning](../strategy/mission-and-positioning.md) — understand what RP is and isn't
- [ ] Read [Who We Are](who-we-are.md) — the identity stack and core beliefs
- [ ] Read [How We Work](how-we-work.md) — client journey, service tiers, decision-making principles
- [ ] Watch or listen to any existing client session recordings (if available and with permission)

**By end of Day 1, you should be able to answer:** What does RP do? What problem does it solve? What makes it different?

---

## Day 2: Learn the Methodology

- [ ] Read [Four Phases](../learning/four-phases.md) — the full methodology deep-dive
- [ ] Read [Facilitation Principles](../learning/facilitation-principles.md) — the five core principles and the facilitation protocol
- [ ] Read [Bilingual Dashboard](../learning/bilingual-dashboard.md) — how we measure in two registers
- [ ] Read [Camera-Shipping Model](../learning/camera-shipping-model.md) — why and how we ship cameras

**By end of Day 2, you should be able to answer:** What are the four phases? What's the difference between Royal Metrics and Nomadic Indicators? Why do we ship cameras instead of sending crews?

---

## Day 3: Learn the Operations

- [ ] Read [Engagement Lifecycle](../operations/engagement-lifecycle.md) — the full SOP from first contact to offboarding
- [ ] Read [Pricing Guardrails](../operations/pricing-guardrails.md) — non-negotiable pricing rules
- [ ] Read [Capacity Management](../operations/capacity-management.md) — hard limits and energy principles
- [ ] Read [Client Communication Standards](../operations/client-communication.md) — how we write and talk to clients

**By end of Day 3, you should be able to answer:** What's the floor rate? What are the capacity limits? What do we send at each stage of the client journey?

---

## Day 4: Set Up the Tools

- [ ] Get access to the Obsidian vault (request from Isaac)
- [ ] Install the CLI tools (`install.sh` from the toolkit repo)
- [ ] Run `rp-pipeline` to see the current prospect pipeline
- [ ] Review the Claude skills list in [Tools & Systems](tools-and-systems.md)
- [ ] Set up DaVinci Resolve if you'll be doing production work

**By end of Day 4, you should have:** All tools installed and working, access to the vault, understanding of where things live.

---

## Day 5: Context & Strategy

- [ ] Read [Theory of Change](../strategy/theory-of-change.md)
- [ ] Read [Competitive Landscape](../strategy/competitive-landscape.md)
- [ ] Read [Phase Roadmap](../strategy/phase-roadmap.md) — where RP is in its multi-year plan
- [ ] Read [Decision Frameworks](../operations/decision-frameworks.md) — how we evaluate opportunities
- [ ] Review [Brand Identity](../strategy/brand-identity.md) — voice, visual identity, design principles
- [ ] Skim the [Reading List](../learning/reading-list.md) — not required reading, but useful for intellectual context

**By end of Day 5, you should be able to answer:** Where is RP in its development? Who's the target client? How do we decide what work to take?

---

## Ongoing

- [ ] Review [Financial Operations](../operations/financial-operations.md) when relevant
- [ ] Start building the [Case Study Library](../learning/case-study-library.md) with any completed engagements
- [ ] Attend (or review notes from) the quarterly review

---

## Questions?

If something doesn't make sense or contradicts something else, flag it. The documentation is a living system. Inconsistencies are bugs, not features.
