## Differentiator (in plain language)

Stakeholder engagement most evaluations never achieve. The facilitation background means people who participate in evaluations actually participate, instead of performing participation for the evaluator. This produces:

- Richer qualitative data (stakeholders share what's actually true, not the official version)
- Higher evaluation utilization (participants helped shape the questions, so they use the results)
- Less performative compliance reporting; more decision-driving evidence

Through-line of every engagement, not a methodological add-on.

## What this practice does NOT do

- Surveillance evaluation (extractive, performative, decoration-only)

## Methodological touchpoints with RP

The RP-canonical Nomadic Indicators Codebook is available as a private qualitative-coding tool but **is not described to evaluation clients as part of the methodology**. When evaluation work calls for qualitative coding:

- Use the standard discipline: codebook → first-pass coding → pattern identification → synthesis → integration
- Build a project-specific codebook grounded in evaluation questions, not in the Nomadic Indicators framework
- Describe the work in plain language: "thematic analysis," "qualitative coding," "constant comparison method"
- The Nomadic Indicators Codebook is a private practice asset that informs HOW Isaac codes, not a deliverable named to the client
