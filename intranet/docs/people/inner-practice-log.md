# Inner Practice Log

The inner practice — meditation, state-work, somatic regulation, shadow work, gene-keys tracking, dark-night navigation — requires its own record. Not therapy notes. Not a journal. A **minimal somatic log** that lets Isaac see patterns across weeks and months without turning the practice into performance.

---

## The system

**Private Obsidian vault section.** Inside the existing `the-metabolizer` vault, a new private folder:

```
the-metabolizer/
  _Inner/             ← private, never synced to Gumroad export
    Daily.md          ← appended to daily, one line
    Weekly.md         ← 5-10 lines once a week
    States/           ← named state-work notes (ad-hoc)
    Shadow/           ← gene-keys shadow entries
```

**Why inside the-metabolizer vault:** the somatic-first practice of Body Vocabulary already lives here. The vault already has the decay + friction-log scaffolding that fits inner work. One tool, one vault.

**Why a `_Inner` folder with underscore:** sorts to the top in Obsidian, visually separates from public vault content, and the underscore prefix is a convention used in the Gumroad export build to exclude it. Never ships.

---

## Daily entry (1 line, 30 seconds)

Appended to `_Inner/Daily.md`. Format:

```
2026-04-04 | [body word] | [state word] | [one-line residue]
```

Examples:
- `2026-04-04 | contracted throat | threshold-alert | something from yesterday's session unmetabolized`
- `2026-04-05 | porous belly | metabolizing | real rest today`
- `2026-04-06 | held breath | avoidance-stillness | opening email to not do the writing`

**Vocabulary:** draw from `System/Body Vocabulary.md` composite states. One word per field. Not sentences.

**When:** end of day, before closing devices. 30 seconds.

---

## Weekly entry (5–10 lines, 5 minutes)

`_Inner/Weekly.md`. Appended Sunday evening:

```markdown
## 2026-W14

**Arc:** [one sentence — what the week moved through]
**Shadow visited:** [gene-key / pattern name]
**Gift accessed:** [if any — don't force it]
**Body's dominant note:** [the most-repeated word from daily entries]
**Residue entering next week:** [what's unprocessed]
**One question for next week:**
```

**Why weekly instead of more:** the daily is for signal capture; the weekly is for pattern recognition. More frequent reflection turns into performance.

## State-work notes (ad-hoc)

When a specific state needs longer attention (not daily, not weekly — episodic):
- New file in `_Inner/States/` named by date + state word
- Free-form, no template
- Use for: sessions with the psychotherapeutic-coach skill, meditation insights worth holding, dream work, recurring somatic pattern tracking

---

## Shadow / Gene Keys tracking

`_Inner/Shadow/` holds one file per gene key being worked:
- File name: `gk-{number}-{name}.md`
- Content: shadow expressions observed, gift moments noticed, siddhi glimpses (rare), connections to current life events
- Append-only. Don't rewrite old entries.

---

## Decay + friction (applied to the inner log)

The Metabolizer's decay principle applies here too:
- **State notes** older than 90 days get reviewed quarterly: metabolize into a weekly entry pattern-observation, or release.
- **Friction log:** if the inner log becomes the avoidance (performing practice instead of practicing), log it in the public `System/Friction Log.md` with a note that the meta-practice caught itself.

---

## Privacy rules

- `_Inner/` never syncs to git (add to .gitignore)
- Never ships in any Metabolizer export
- Never published, quoted, or referenced in public content
- If a pattern from the inner log becomes something Isaac wants to publish about, it gets fully rewritten in a public note first — the inner log entries themselves stay private

---

## The underlying stance

The inner practice is the ground the outer practice grows from. Without a minimal record, patterns that repeat across months become invisible. With too much record, the log becomes the practice. This cadence — 30 seconds daily, 5 minutes weekly, ad-hoc when needed — is the thinnest version that still reveals arc.

*"Log the practice enough to see the arc. Not enough to replace it."*
