# Training Periodization — Full Embodied Practice

Running has implicit periodization (race cycles → Base → Build → Peak → Taper → Recovery). The other embodied practices — meditation, mobility, strength, creative work — have historically been reactive. This doc applies the periodization model across all practice domains.

The principle: **practice cycles in phases, not at a constant volume.** A life that sustains ultra-distance effort, intensive facilitation work, and creative output without collapse requires phase logic applied to all of it, not just running.

---

## The phases (borrowed from ultra training)

| Phase | Duration | Character | What it builds |
|---|---|---|---|
| **Base** | 6–10 weeks | High volume, low intensity, patient | Aerobic/capacity floor |
| **Build** | 4–6 weeks | Moderate volume, rising intensity | Specific fitness |
| **Peak** | 2–3 weeks | Lower volume, high intensity/specificity | Sharpness |
| **Taper** | 1–2 weeks | Low volume, low intensity | Arrival |
| **Race / Output** | 1 week | Full expression | The actual thing |
| **Recovery** | 2–4 weeks | Minimal structure, restoration | Adaptation, integration |

Full cycle: ~16–26 weeks. Two to three cycles per year is the realistic ceiling for any single domain.

---

## Applied across domains

### Running
Already periodized. Calendar organized around A-races. **No change needed.**

### Creative / facilitation work
Treat major engagements (Organizational Embedding retainers, film productions, Say The Thing intensives, multi-site evaluation kickoffs) as **races**. Treat sustained writing/research windows as **races**.

| Race | Example |
|---|---|
| A-race | An Organizational Embedding engagement kickoff; a film shoot week; a public workshop; an evaluation site visit |
| B-race | A Program Engagement kickoff; an evaluation proposal submission; a newsletter issue; an essay draft |
| C-race | Weekly publishing; a Founder Story session; a facilitation session in an ongoing engagement |

**Base → Build → Peak → Race cadence** for an A-race creative engagement:
- Base (4 weeks out): research, drafts, client listening, capacity protection
- Build (2 weeks out): structure forming, narrative shape, dry-run facilitations
- Peak (1 week out): tight rehearsal, energy conservation
- Race: the engagement itself
- Recovery (2 weeks after): no new creative commitments, writing is optional, rest centered

### Strength / mobility
Runs as **counter-cyclic** to running:
- Running Base phase → strength Build phase (strength volume high)
- Running Peak phase → strength Maintenance (light, injury-prevention only)
- Running Recovery → strength Base (rebuild load)

### Meditation / inner practice
Does not periodize by volume (daily practice is constant). **Periodizes by depth:**
- Running Base = inner practice Base (standard daily)
- Running Build = inner practice Build (add longer weekly sits, shadow work)
- Running Peak/Race = inner practice Maintenance (protect daily, don't add load)
- Running Recovery = inner practice Deep (retreats, extended sits, dream work)

Rule: **no two domains at Peak simultaneously** for more than 3 weeks. If they must overlap, one domain yields.

## Coordination rules

1. **Running Peak + Creative Peak = never.** A-race running overlapping an A-race engagement produces injury or mediocre work. Schedule around.
2. **Recovery after every A-race** is non-negotiable. 2 weeks minimum. This is where the adaptation happens.
3. **One A-race per domain per block.** Two A-race creative engagements in one 6-week window is a capacity violation.
4. **Sabbatical week = mandatory Recovery week** for all domains. No exceptions.
5. **Log phase transitions** in activities.json or inner-practice log. Pattern tracking reveals whether the phase shape is working.

---

## Signals the periodization is working

- Injuries decrease (running + strength cross-managed)
- Creative output has fewer "I'm empty" weeks (Recovery phases do their work)
- Facilitation quality stays high in A-race engagements (Peak phase actually peaks)
- Sabbatical rhythm holds (phase logic protects it)
- Delegation signals stay cold (see `delegation-signals.md`)

## Signals it's NOT working

- Multiple Peak phases running simultaneously (failure mode #1)
- Recovery phases shortened or skipped (failure mode #2)
- Same domain in Build for more than 8 weeks (it should have raced or recovered by now)
- Inner practice falls to zero during Creative Peak (meditation is Maintenance, not Abandon)

---

## Review cadence

- **Weekly:** which phase is each domain in? Log in weekly threshold check.
- **Quarterly:** review the phase map against actual experience. Adjust the annual template.
- **Annual:** redesign the template based on the year's race calendar + engagement bookings.
